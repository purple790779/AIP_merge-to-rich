# 에셋 생성 및 관리 가이드

상태: Current  
기준일: 2026-03-08

본 문서는 게임 내 이미지, 아이콘, 그래픽 에셋을 추가하거나 교체할 때의 기준을 정리합니다.

## 1. 에셋 폴더 구조

```text
public/
├── favicon.ico
├── icon.png
├── assets/
│   └── coins/
└── ...
```

## 2. 에셋 소스 원칙
- 출처가 불분명한 이미지를 그대로 가져와 쓰지 않습니다.
- AI 생성 에셋은 빠른 시안 제작에 유용하지만, 저작권과 정책 리스크가 자동으로 사라지는 것은 아닙니다.
- 최종 반영 전에는 아래를 검토해야 합니다.
  - 상표/로고 유사성
  - 특정 게임/브랜드의 트레이드드레스 유사성
  - 과도하게 익숙한 캐릭터/아이콘 형태
  - 스토어 정책 위반 가능성

## 3. AI 에셋 요청 가이드
AI에게 요청할 때는 아래 정보를 같이 주는 편이 좋습니다.
- 파일 경로
- 해상도
- 투명 배경 여부
- 스타일 키워드
- 사용 위치(UI 아이콘 / 코인 / 스토어 그래픽 등)

예시:

> "17레벨 코인 시안을 만들어줘. 512x512 투명 PNG 기준이고, 기존 코인보다 희귀도가 높아 보이되 현재 게임 톤과 어긋나지 않게 부탁해. 저장 경로는 `public/assets/coins/coin_17.png`로 맞춰줘."

## 4. 반영 전 체크리스트
- 게임 안 다른 에셋과 시각 톤이 맞는지
- 작은 화면에서도 식별 가능한지
- 레벨/희귀도 구분이 되는지
- 브랜드/저작권 리스크가 없는지

## 5. 자동화 스크립트

### `generate-android-icons.cjs`
- 목적: 마스터 아이콘 한 장으로 Android 해상도별 아이콘 생성
- 사용 시점: 앱 아이콘 교체 또는 신규 프로젝트 파생 시

### 스토어 그래픽 생성 스크립트
- 플레이 스토어용 그래픽 자동 생성 보조
- 최종 제출 전에는 결과물을 수동 검수해야 합니다.
