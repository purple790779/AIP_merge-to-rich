# 에셋 생성 및 관리 가이드 (ASSET_GUIDE)

본 문서는 게임 내 이미지, 아이콘, 그래픽 에셋들을 관리하고 새롭게 생성하는 방법을 설명합니다. 특히 저작권 문제 없이 빠르고 안전하게 에셋을 추가하기 위해 **AI 에이전트를 적극 활용하는 지침**을 포함하고 있습니다.

## 1. 에셋 폴더 구조
모든 정적 이미지 파일은 `public/` 폴더 내부에 위치합니다.

```text
public/
├── favicon.ico              # 웹 브라우저 탭 아이콘 설정
├── icon.png                 # PWA(웹앱)용 기본 아이콘 (512x512 권장)
├── assets/                  # 게임 내 사용 에셋 폴더
│   └── coins/               # 머지용 동전 텍스처 폴더 (coin_1.png ~)
└── ...
```

## 2. 🤖 AI 에이전트를 활용한 에셋 생성 가이드 (중요)

구글 플레이나 토스 미니앱 등 공식 스토어에 앱을 출시할 때, 구글 링이나 외부 에셋 스토어에서 퍼온 이미지들은 **저작권(Copyright) 검열 문제로 앱 강제 삭제 또는 리젝의 원인**이 됩니다.
향후 이 프로젝트나 후속 프로젝트를 개발할 때, 디자인 작업은 **본 프로젝트를 코딩해주는 AI 에이전트(저에게) 직접 요청하는 것을 원칙**으로 합니다.

### AI에게 에셋 생성을 요청하는 프롬프트(명령) 예시
코딩을 돕고 있는 AI 에이전트에게 다음과 같이 명확한 '규격'과 '스타일'을 지정해서 에셋 이미지를 만들어 달라고 요청하십시오.

> **예시 1 (신규 코인 추가 시)**:
> "다음 레벨인 17레벨 동전 이미지를 만들어줘. 크기는 512x512 픽셀이고, 배경이 없는 투명 PNG 텍스처 파일이어야 해. 스타일은 약간 입체감 있는 카툰 렌더링 스타일의 반짝이는 다이아몬드 코인으로 부탁해. 완성된 이미지는 `public/assets/coins/coin_17.png` 경로에 저장해줘."

> **예시 2 (새로운 인게임 이펙트/UI 추가 시)**:
> "상점에서 사용할 폭죽 터지는 아이콘 에셋을 하나 그려줘. 256x256 크기의 투명 PNG 텍스처로 생성해주고 픽셀아트 스타일로 해줘."

AI가 직접 에셋 파일(`.png`, `.webp` 등)을 생성하여 올바른 폴더에 즉시 삽입해 주므로, 에셋 스토어나 포토샵을 열 필요가 없습니다. 완전 창작물이기 때문에 저작권에서도 100% 프리패스입니다.

## 3. 스토어 출시용 에셋 스크립트 도구 (자동화)

저희가 구현한 `node.js` 기반 에셋 스크립트를 사용하면, 구글 플레이 스토어 제출이나 안드로이드 빌드용으로 여러 장의 잘린/조정된 이미지를 손수 만들 필요가 없습니다.

### `generate-android-icons.cjs`
- **목적**: 하나의 512x512 마스터 앱 아이콘 통짜 리소스를 기반으로, 안드로이드 전용 해상도 아이콘들(`mipmap-mdpi`, `hdpi` 등 수십 개)을 자동 생성합니다.
- **사용법**: AI에게 의뢰하여 메인 아이콘 파일로 쓸 원본 이미지 파일(`app_icon_final_512.png` 등)을 받은 후, 터미널에 `node generate-android-icons.cjs`를 입력합니다. (단, 이 명령어는 AI에게 바로 실행해달라고 명령하면 알아서 다 해줍니다.)

### `generate-feature-graphic.cjs` 등
- 플레이 스토어 설명 페이지에 들어갈 가로형(1024x500) 그래픽 스크린샷 텍스처를 자동 생성하는 스크립트들입니다. 필요 시 코드를 커스터마이징하여 쓸 수 있습니다.
