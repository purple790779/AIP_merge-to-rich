# 머지 머니 타이쿤 - Phase 1 라이브 운영 기획서

상태: Planned  
기준일: 2026-03-08

이 문서는 라이브 운영 기준의 Phase 1 계획서입니다. 이미 구현된 항목은 현황으로 적고, 아직 남은 일만 작업 후보로 남깁니다.

## 1. 현재 전제
- 게임은 출시 이후 라이브 운영 단계입니다.
- 핵심 게임 루프와 기본 상점/도감/부스트는 구현되어 있습니다.
- `v1.4.3` 기준으로 일일 보상은 구현 완료 상태입니다.
- 복귀 보상은 설계 완료 상태이며 다음 구현 후보입니다.

## 2. 이번 Phase 1의 목표

### 목표 A — 안정성 유지
- 저장/로드 회귀 방지
- 세이브 호환성 확인
- 빌드 및 주요 UI 흐름 안정성 유지

### 목표 B — 리텐션 구조 보강
- 일일 보상 이후 다음 재접속 동기를 추가
- 단순 콘텐츠 추가보다 반복 접속 동기와 복귀 체감을 강화

### 목표 C — 후속 메타 준비
- 환생/투자/오프라인 보상 등 중장기 구조의 설계 기반 마련

## 3. 현황 요약

### 구현 완료
- 일일 보상
- 도감
- 수익 배율 업그레이드
- 자동 병합 속도 업그레이드
- 기본 업적/부스트/보석 시스템

### 설계 완료, 구현 전
- 복귀 보상

### 미구현
- 오프라인 보상
- 일일/주간 미션
- 환생(Prestige)
- 투자 시스템

## 4. 이번 Phase 1의 실제 작업 후보

### 4.1 복귀 보상 구현
- `RETURN_REWARD_PLAN.md` 기준으로 store 상태/액션 추가
- 앱 진입 시 eligibility 계산
- 복귀 보상 모달 및 지급 처리
- 악용 방지와 중복 수령 차단

### 4.2 오프라인 보상 최소형 설계
- 종료 후 경과 시간 기반 최소 정산 구조
- 과도한 보상보다 복귀 체감 우선
- 리텐션 기능과 충돌하지 않는 범위로 설계

### 4.3 미션형 목표 시스템 초안
- 일일/주간 미션 후보 정의
- 현재 업적과 역할이 겹치지 않게 분리

### 4.4 문서/QA 유지
- 체인지로그와 로드맵 최신화
- 저장/로드/자정 경계 QA 기록

## 5. 기술 우선순위
- 신규 상태 필드는 `GameState`, `initialState`, `partialize`, `onRehydrateStorage` fallback까지 같이 반영
- 날짜 기준 기능은 KST로 통일
- 수치형 보상은 초기엔 보수적으로 적용
- 구현 전후로 빌드/세이브/중복 수령 회귀 체크 필수

## 6. 작업 원칙
- 로컬에서 충분히 검증한 뒤 반영
- 작은 단위 커밋 유지
- 문서와 실제 구현 상태를 같이 업데이트
