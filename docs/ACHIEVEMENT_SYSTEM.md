# 업적 시스템 가이드

상태: Current  
기준일: 2026-03-09

## 개요
- 정의 파일: `src/game/achievements.ts`
- 총 업적 수: 108개
- 구성: 7개 카테고리(발견/머지 숙련/부의 확장/수집/복귀 루프/업그레이드 숙련/완성자)
- 보상 지급: 자동 해금 시 즉시 지급(`checkAchievements` 경로)

## 카테고리 의도
- 발견: 코인 티어 개척 자체를 장기 동기로 유지
- 머지 숙련: 플레이 반복에서 누적되는 실행력의 단계적 성취 제공
- 부의 확장: 보유 자산 축 + 누적 수익 축을 동시에 강화
- 수집: 도감 폭과 보드 점유 밀도를 분리된 목표로 제시
- 복귀 루프: 일일/복귀/오프라인 루프를 숙제가 아닌 선택형 운영으로 설계
- 업그레이드 숙련: 생산/수익/자동화 업그레이드의 최적화 단계를 명확히 노출
- 완성자: 로드맵 완주 + 도전 체인 + 엔드게임(전 업그레이드/최대 자산) 집중

## 레거시 랭크
- 업적 해금 시 티어별 점수를 획득하고 누적 점수로 레거시 랭크가 상승합니다.
- 랭크는 저장 상태를 추가하지 않고 해금 업적 목록에서 계산됩니다.
- 최상단 랭크 카드에 현재 랭크/점수/다음 랭크 진행률을 표시합니다.

## UI 흐름 (카테고리 2단 구조)
1. 1차 화면: 상위 카테고리 카드만 노출
2. 카테고리 선택: 해당 카테고리 세부 업적 리스트로 전환
3. 세부 화면: 잠금/해금/진행도/보상을 확인하고, 상단 Back 버튼으로 카테고리 목록 복귀

## 상세 화면 UX 안정화 메모
- 상세 헤더는 현재 sticky 대신 일반 레이아웃으로 유지해 모바일에서 레이어 충돌을 피합니다.
- 잠금 업적은 카드 전체 opacity를 낮추지 않고, 배경/테두리/텍스트 대비를 조정하는 방식으로 구분합니다.
- 상세 리스트는 단일 스크롤 컨테이너 기준으로 유지하고, 카드 터치 영역은 전 구간에서 동일하게 반응하도록 관리합니다.

## 운영 원칙
- 신규 업적 확장 시 상태 필드 추가보다 기존 `GameState` 메트릭 활용을 우선합니다.
- 보상 지급 경로는 `checkAchievements` 단일 자동 해금 흐름을 유지합니다.
- 미션은 수령형 루프, 업적은 자동 해금형 장기 동기로 역할을 분리합니다.
