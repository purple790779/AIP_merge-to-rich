## 랭크 메타
- 업적은 tier별 점수를 가진다
- 누적 점수로 랭크가 계산된다
- 업적 메뉴 상단에서 현재 랭크 / 점수 / 다음 랭크 진행도를 보여준다

## UI 구조
1. 1차 화면: 카테고리 카드 목록 + `지금 신경 쓰면 좋은 카테고리` spotlight 요약
2. 카테고리 선택: 해당 카테고리 상세 목록
3. 상세 화면: 필터 카드(전체 / 진행 중 / 완료 / 미진행) + 추천 업적 요약 + 상태별 카드 리스트

## 상세 화면 UX 원칙
- 두 단계 탐색 구조는 유지하되, 카테고리 목록에서 `근접` / `진행` / `완주` pill로 현재 집중 대상을 먼저 읽게 한다
- `spotlightCategories`는 저장 데이터를 바꾸지 않고 현재 진행률만으로 곧 달성 가능한 카테고리 2개를 추천한다
- `selectedRecommendedId`는 잠금 업적 중 진행률, 남은 수치, 기존 진척 유무를 함께 비교해 `다음 추천 업적`을 고른다
- `전체` 필터는 정의 순서를 버리지 않으면서도 새 업적 / 추천 업적 / 근접 업적 / 진행 중 업적을 먼저 보여줘 현재 집중 대상을 빠르게 찾게 한다
- 잠금 업적은 카드 전체 opacity가 아니라 배경/테두리/상태 pill 대비로 구분한다
- 업적 카드는 설명을 2줄 이내로 정리하고, 진행도 / 남은 수치 / 상태 메모를 하단에 배치해 한 번에 훑을 수 있게 한다
- 상세 리스트는 단일 스크롤 컨테이너 기준으로 유지한다
- 업적 메뉴는 열 때 `checkAchievements()`를 다시 호출하지 않는 `view-first` 구조다

## 운영 메모
- 업적은 자동 해금, 미션은 수동 보상 수령으로 역할을 분리한다
- 새 업적 추가 시 가능하면 기존 `GameState` 메트릭을 우선 재사용한다
- 카테고리 메타(`ACHIEVEMENT_CATEGORY_META`)는 아이콘 / 타이틀 / 서브카피 / 강조색을 함께 관리하므로 UI 톤 수정 시 우선 점검한다
