# 업적 시스템 가이드

상태: Current  
기준일: 2026-03-09

## 개요
- 정의 파일: `src/game/achievements.ts`
- 총 업적 수: 64개
- 구성: 7개 카테고리(발견/머지 숙련/부의 확장/수집/복귀 루프/업그레이드 숙련/완성자)
- 보상 지급: 자동 해금 시 즉시 지급(`checkAchievements` 경로)

## 카테고리 의도
- 발견: 최고 코인 레벨 개척 루프를 장기 동기로 유지
- 머지 숙련: 반복 플레이에서 자연스럽게 누적되는 핵심 실행력 동기
- 부의 확장: 보유 자산 + 누적 수익 축으로 성장 체감 강화
- 수집: 도감 발견과 보드 점유를 별도 목표로 분리
- 복귀 루프: 일일/복귀/오프라인 루프를 "숙제"보다 "선택적 유지"로 설계
- 업그레이드 숙련: 생산 라인 최적화 진행 상황을 다단계로 제시
- 완성자: 성장 로드맵 완주, 전 업그레이드 맥스, 최대 자산 같은 장기 엔드게임 목표

## 레거시 랭크(경량 메타)
- 업적 해금 시 티어별 점수를 획득하고, 누적 점수로 레거시 랭크가 상승합니다.
- 랭크는 UI 정보 레이어이며 저장 안정성에 영향을 주지 않습니다(해금된 업적 목록에서 계산).
- 최상단 랭크 카드에 현재 랭크/점수/다음 랭크 진행률을 표시합니다.

## UI 구성
- `AchievementModal` 상단: 전체 해금률 + 레거시 랭크 카드
- 카테고리 스트립: 카테고리별 완료 수(예: 4/9)
- 카테고리 섹션: 업적 리스트 + 잠금 상태 업적의 실시간 진행도(숫자/퍼센트)

## 운영 원칙
- 업적은 장기 동기를 담당하고, 일일 퀵윈은 보조 루프로 유지합니다.
- 업적 확장 시에는 신규 상태 필드를 최소화하고, 기존 저장 데이터(`GameState`) 기반 계산을 우선합니다.
- 보상 밸런스 조정 시 `checkAchievements` 단일 지급 경로를 유지합니다.
