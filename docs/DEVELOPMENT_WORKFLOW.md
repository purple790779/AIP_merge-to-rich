## 2. 제작 방법 (품질 기준)

이 프로젝트는 단순히 기능을 많이 붙이는 방식으로 진행하지 않습니다.

### 핵심 원칙
1. **구조 우선**
   - 기능 추가보다 먼저 플레이 흐름, 정보 계층, 화면 구조, 확장성을 검토합니다.
2. **모바일 우선**
   - 실제 플레이는 좁은 화면에서의 체감이 중요하므로, 보드 가시성과 터치 UX를 최우선으로 둡니다.
3. **그래픽 / 분위기까지 포함한 완성도 관리**
   - 카드, 버튼, 배지, 여백, 색상, 강조 흐름까지 게임다운 감각으로 맞춥니다.
4. **장기 플레이 동기 강화**
   - 숙제형 압박 대신, 업적/미션/월드 진행/투자/보상 구조가 자연스럽게 다음 목표를 제시해야 합니다.
5. **저장 안정성 우선**
   - 라이브 운영 단계인 만큼, 세이브 호환성 / 보상 중복 방지 / 리셋 규칙 일관성을 항상 우선합니다.

### 구현 우선순위
- 1순위: 실제 플레이 체감 개선
- 2순위: 장기 진행 구조 강화
- 3순위: 그래픽/레이아웃/가독성 폴리시
- 4순위: 문서/인수인계/QA 기준선 유지

### 검증 기준
코드 변경이 의미 있게 발생하면 기본적으로 아래를 확인합니다.

```bash
npm run lint
npm run build
```

가능한 경우 다음도 함께 확인합니다.
- 좁은 화면에서의 버튼/카드 겹침 여부
- 지역/업적/미션/상점/일일 보상 간 상호작용
- KST 기준 리셋/연속 보상/streak 표시 일관성

## 3. 멀티에이전트 운영 방식

이 프로젝트는 필요할 때 **메인 총괄 + 하위 2~3 에이전트** 구조로 운영합니다.

### 메인 역할
메인은 아래를 담당합니다.
- 전체 방향 설정
- 우선순위 조정
- 브랜치/워크트리 관리
- 결과 취합 및 통합
- 최종 검증 / 최종 보고

### 하위 에이전트 역할 예시
#### Atlas
- 메인 셸 / 월드 / 지역 / 화면 구조 / 모바일 레이아웃
- 보드 중심 레이아웃, 지역 진행감, 월드 여정 연출 강화

#### Lyra
- 업적 / 미션 / 상점 / 진행 유도 UX
- 추천 대상, 근접 목표, 다음 행동 신호, 카드 가독성 강화

#### Rook
- 보상 / 밸런스 / 문서 / QA / handoff
- 일일 보상, 복귀 보상, 경제 밴드, 검증 기준선, 문서 정리

## 4. 멀티에이전트 작업 원칙

### 4-1. Worktree 분리
충돌을 줄이기 위해 에이전트마다 별도 worktree / 브랜치를 사용합니다.

예시:
- `feat/...-shell-*`
- `feat/...-ux-*`
- `feat/...-balance-*`

### 4-2. 역할 충돌 최소화
- Atlas는 셸/레이아웃/월드 중심
- Lyra는 진행 UX/모달/상점 중심
- Rook은 밸런스/문서/QA 중심

즉, 같은 파일을 동시에 마구 수정하지 않도록 역할을 분리합니다.

### 4-3. 메인 브랜치 통합 규칙
하위 에이전트 결과는 바로 main에 넣지 않고,
먼저 메인 작업 브랜치에 취합합니다.

통합 순서 예시:
1. 하위 브랜치 작업 완료
2. 각 브랜치 lint/build 확인
3. 메인 작업 브랜치로 cherry-pick 또는 통합
4. 충돌 정리
5. 메인 작업 브랜치 최종 lint/build
6. 그 후 PR / push / merge 판단

### 4-4. 중간보고 / 문서 업데이트
작업 중에는 다음을 유지합니다.
- 중간보고
- 체크포인트 커밋
- 진행 로그 기록
- changelog / architecture / roadmap / handoff 동기화

기준 진행 로그 파일:
- `docs/AUTONOMOUS_PROGRESS_2026-03-10.md`

## 5. “대충하지 않기” 운영 규칙

이 프로젝트는 다음 기준을 명시적으로 적용합니다.

- “기능만 되면 됨” 금지
- 보기만 화려한 가짜 깊이 금지
- 보드보다 UI가 더 시끄러워지는 구조 금지
- 모바일에서 읽기 힘든 과밀 정보 배치 금지
- 문서와 코드가 따로 노는 상태 금지
- 검증 없이 완료 처리 금지

즉, **베테랑 게임 개발자 / 기획자 / UI 디자이너 기준**으로,
- 구조
- 그래픽
- 정보 계층
- 장기 플레이 동기
- 저장 안정성
을 함께 관리합니다.
